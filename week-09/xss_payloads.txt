==================================================
SESSION 26: CLIENT-SIDE ATTACK PAYLOADS
==================================================

1. REFLECTED XSS
* Payload Used: <script>alert('XSS')</script>
* Concept: Why does this attack disappear when you refresh the page?
Answer: Reflected XSS is non-persistent; the script is only executed when it is part of the URL parameters for that specific request.

2. STORED XSS (Cookie Theft)
* Cookie Value Stolen: session_id=tracked_via_payload
* Payload Used: <script>alert(document.cookie)</script>

3. CROSS-SITE REQUEST FORGERY (CSRF)
* Malicious URL Crafted: [http://127.0.0.1:8081/transfer?to=Attacker&amount=5000](http://127.0.0.1:8081/transfer?to=Attacker&amount=5000)
* Concept: What is the primary defense against CSRF?
Answer: The primary defense is the use of Anti-CSRF Tokens (also known as synchronizer tokens). These are unique, unpredictable secrets generated by the server for every user session or request, which the client must include in any state-changing request (like a transfer) to prove it was intentional and not forged by a third party.
